# S11 — File detail (summary view)

**Personas:** All external and in-house practitioners
**User stories:** FR-B1-325, FR-B1-327, FR-B1-319, FR-B1-320, FR-B1-321

## Purpose

Single source of truth for a grant: scope, commitment, conditions, correspondence, documents, invoices, outcomes.

## Layout

SLDS record-detail pattern: highlights panel at top, tabbed detail below, related lists on the right.

## Elements to include

- Highlights panel: Client, Q-number, Matter type, Status badge, Scope value, Conditions pill
- Tabs: Summary, Correspondence, Documents, Invoices, Outcomes, Activity history
- Summary tab: scope, commitment, conditions, key dates, assigned LAQ contact
- Quick actions: Request extension, Submit invoice, Upload document, Send message
- Right rail: decision letters (most recent first), assigned parties, notifications

## States

Active, archived, conditions pending, awaiting LAQ response.

## Designer notes

The summary is the single hardest-working screen in the portal. Get hierarchy right: status, scope, next action visible without scrolling.
